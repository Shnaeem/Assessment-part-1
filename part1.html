Part 1: 30%, you must create a repo for part 1, separate from part 2 1. Why does React utilize the virtual-dom? Ans. Virtual DOM enhances the performance that is why react uses virtual DOM. In a real DOM the whole pages is loaded every time the data is changed but in virtual dom only the part of the page where the change has occured will be loaded. A virtual dom is a light weight Javascript object which is basically copying the real dom. So basically whenever any underlying data changes, the entire UI is re-rendered in virtual Dom then the difference between the previous DOM representation and the new one is calculated. Once the calculations are done, the real DOM will be update with only the things that have actually changed. 2a. How is data flowed in React? Ans. Why would a startup be interested in applying React-Routing to a project? Explain in detail. Ans. The performance on the frontend of a website is directly affected by things like the number of pages loaded initially, the amount of data fetched and displayed, and the time taken to switch from one page to another. React routing offers following advantages, - Routing between components is fast as the amount of data that renders is less. The rest of the data is rendered by the DOM, and even when there's tons of HTML and CSS to render, the DOM handles that part in the blink of an eye. Using lazy loading, any delay in rendering HTML is compensated for. - For better user experience, animations and transitions can be easily implemented when switching between different components. - It gives a real sense of a single-page application in action. No separate pages are rendered, and the current page doesn't refresh to load a new view. In short, React routing offers better performance and speed. 5. Can browsers read JSX? Why? Ans.
